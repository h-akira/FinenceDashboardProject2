# Enhancement ワークフロー

改善・仕様追加・仕様変更を4フェーズで実施する。

## ワークフロー概要

```
フェーズ1: 起票・計画
  ├── docs/ と実装を確認して現状を把握
  ├── enhancements/XXX_*.md を作成（テンプレート準拠）
  └── 開発者にレビュー提示

フェーズ2: ドキュメント更新
  ├── docs/ に更新が必要か確認
  ├── 更新があれば開発者の承認を得て更新
  └── docs/history/XXX_*.md に変更通知を作成（テンプレート準拠）

フェーズ3: 実装
  ├── 各ユニットを独立して実装
  └── 各ユニットのテストを実施

フェーズ4: 完了
  └── enhancement ファイルの「対応」セクション追記・ステータス更新
```

## 重要なルール

- **テンプレートに従う**: `enhancements/XXX_template.md` と `docs/history/XXX_template.md` を必ず読み、書式と AI instruction コメントの指示に従う
- **採番**: 既存ファイルの最大番号 + 1 を使用する
- **docs/ は正（Single Source of Truth）**: 実装は `docs/` の定義に厳密に従う
- **docs/ の変更には許可が必要**: 共有ドキュメントの変更前に開発者に確認する
- **docs/ 変更時は通知必須**: `docs/history/` に変更通知ファイルを必ず作成する
- **実装前にドキュメントを読む**: 関連する `docs/` を読まずに実装を開始しない
- **他サブモジュールの内部実装に依存しない**: ユニット間は `docs/` の契約で連携する
- **ドキュメント間の整合性を維持する**: `docs/` 内のドキュメントは相互に参照・依存している。1つのドキュメントを変更する際は、関連する他のドキュメントとの整合性を確認し、矛盾が生じないようにする
