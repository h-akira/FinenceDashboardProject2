# ユーザーストーリー

## 概要

本ドキュメントは、金融ダッシュボードアプリケーション「FinanceDashboardProject2」のユーザーストーリーをまとめたものである。
ユーザーが本アプリケーションで実現できることを網羅的に記述する。

## アクター定義

| アクター | 説明 |
|---------|------|
| 未ログインユーザー | ログインしていない状態のユーザー。アプリケーションの機能は一切利用できない |
| ログインユーザー | Cognito 認証を経てログイン済みのユーザー |

> アカウントの作成は管理者がマネジメントコンソールまたは CLI で行う。セルフサインアップは提供しない。

---

## 1. 認証

### US-1.1: ログイン

- **アクター**: 未ログインユーザー
- **ストーリー**: ユーザーとして、自分のアカウントにログインしたい。ダッシュボードの各機能を利用するためである。
- **受け入れ条件**:
  - 未ログイン状態でアプリケーションにアクセスすると、ログイン画面（Cognito Managed Login） に自動的にリダイレクトされる
  - 認証成功後、コールバック処理を経てダッシュボードページが表示される
  - ヘッダーにユーザー名が表示され、ナビゲーションが利用可能になる

### US-1.2: ログアウト

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、ログアウトしたい。セッションを終了し、セキュリティを確保するためである。
- **受け入れ条件**:
  - ヘッダーの「ログアウト」ボタンをクリックするとログアウト処理が実行される
  - ログアウト後、未ログイン状態のページにリダイレクトされる

### US-1.3: パスワード変更

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、パスワードを変更したい。アカウントのセキュリティを維持するためである。
- **受け入れ条件**:
  - プロフィールページからパスワード変更画面に遷移できる
  - 現在のパスワードと新しいパスワードを入力して変更できる
  - 変更成功後、完了メッセージが表示される

### US-1.4: 未ログイン時のアクセス制限

- **アクター**: 未ログインユーザー
- **ストーリー**: 未ログイン状態ではアプリケーションの機能を一切利用できないようにしたい。セキュリティを確保するためである。
- **受け入れ条件**:
  - 未ログイン状態でダッシュボードや各ページの URL に直接アクセスしても、ログイン画面（Cognito Managed Login） にリダイレクトされる
  - API エンドポイントは認証なしではアクセスできない

## 2. ユーザー管理

### US-2.1: プロフィールの確認

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、自分のプロフィール情報を確認したい。アカウント情報を把握するためである。
- **受け入れ条件**:
  - プロフィールページに以下が表示される
    - ユーザー名
    - メールアドレス（認証済みの場合はチェックマーク付き）
    - 登録日
  - ヘッダーのユーザー名ボタンからプロフィールページに遷移できる

### US-2.2: アカウントの削除

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、アカウントを削除したい。サービスの利用をやめるためである。
- **受け入れ条件**:
  - プロフィールページの「アカウント削除」ボタンからアカウント削除画面に遷移できる
  - 警告メッセージが表示される
  - パスワードを入力して「アカウントを削除」ボタンをクリックできる（パスワード未入力時はボタンが無効）
  - 最終確認ダイアログが表示される
  - 削除を選択するとアカウントが削除され、ログアウト後に未ログイン状態のページに遷移する

---

## 3. ダッシュボード

### US-3.1: ダッシュボードの表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、ログイン後にダッシュボードを確認したい。金融市場の情報をまとめて閲覧するためである。
- **受け入れ条件**:
  - ダッシュボードページに各チャートウィジェットが表示される
  - 各チャートの表示/非表示をチェックボックスで切り替えられる

### US-3.2: レイアウトの切り替え

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、ダッシュボードのレイアウト列数を切り替えたい。画面サイズや好みに合わせて表示を最適化するためである。
- **受け入れ条件**:
  - デスクトップ表示（大画面）時に、レイアウト列数を 1列 / 2列 / 3列 から選択できる
  - モバイル表示時は自動的に 1列レイアウトになる

## 4. TradingView Widget チャート

### US-4.1: マーケット概況の表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、グローバル市場の主要指数・先物・通貨の概況を一覧で確認したい。市場全体の動向を把握するためである。
- **受け入れ条件**:
  - TradingView マーケット相場ウィジェットが表示される
  - 以下のカテゴリのシンボルが表示される
    - 指数（S&P 500, US 100, Dow 30, Nikkei 225, DAX Index, UK 100）
    - 先物（Gold, Oil, Gas, Corn, US10Y）
    - FX（USD/JPY, EUR/JPY, EUR/USD, GBP/JPY, AUD/JPY, GBP/USD, AUD/USD）

### US-4.2: 経済カレンダーの表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、経済指標の発表予定を確認したい。市場に影響を与えるイベントを把握するためである。
- **受け入れ条件**:
  - TradingView 経済カレンダーウィジェットが表示される
  - 主要通貨（AUD, USD, CAD, EUR, JPY, CHF, GBP 等）の経済指標が表示される

### US-4.3: FX 通貨ペアチャートの表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、FX 通貨ペアのチャートを確認したい。為替の値動きをテクニカル分析するためである。
- **受け入れ条件**:
  - TradingView チャートウィジェットが表示される
  - ドロップダウンで通貨ペアを切り替えられる（USD/JPY, EUR/JPY, EUR/USD, GBP/JPY, GBP/USD, AUD/JPY, AUD/USD, ドル指数）
  - ドロップダウンでテクニカル指標を切り替えられる（一目均衡表 / ボリンジャーバンド+SMA）
  - チャートの高さをドラッグで調整できる

### US-4.4: 日経225チャートの表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、日経225のチャートを確認したい。日本株市場の動向をテクニカル分析するためである。
- **受け入れ条件**:
  - TradingView チャートウィジェットが日経225（JP225USD）で表示される
  - ドロップダウンでテクニカル指標を切り替えられる（一目均衡表 / ボリンジャーバンド+SMA）
  - チャートの高さをドラッグで調整できる

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、日本のマネーサプライ M2 の推移を確認したい。金融政策の影響を把握するためである。

  - 開始日・終了日を日付ピッカーで指定して表示期間を変更できる
  - デフォルトでは直近 10 年間が表示される

## 6. バックエンドデータ チャート

### US-6.1: 政策金利・長期金利チャートの表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、米国の政策金利・長期金利の推移チャートを確認したい。金融環境の把握に役立てるためである。
- **受け入れ条件**:
  - ダッシュボードページに政策金利（FF金利誘導目標）と米国10年国債利回りの推移チャートウィジェットが表示される
  - データはバックエンド API (`/finance/interest-rate`) から取得される
  - 2025年以前のデータは固定で保持され、それ以降のデータのみ外部から取得して結合される
  - ページのリロード時または更新ボタンの押下時にデータが再取得される
  - 更新ボタンが用意されている

### US-7.1: ヘッダーナビゲーション

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、ヘッダーから各ページに移動したい。アプリケーション内を効率的に移動するためである。
- **受け入れ条件**:

    - ダッシュボード
    - カスタムチャート
    - ユーザー名ボタン（クリックでプロフィールページに遷移）
    - ログアウトボタン

## 8. カスタムチャート

### US-8.1: カスタムチャートのソース選択と表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、複数の金融指標を自由に選択してカスタムチャートに表示したい。異なる指標を重ねて比較・分析するためである。
- **受け入れ条件**:
  - カスタムチャートページ (`/custom-chart`) にアクセスできる
  - バックエンドから取得したデータソース一覧がチェックボックスで表示される
  - チェックボックスでソースを選択し、反映ボタンを押すとデータが取得されチャートに描画される
  - チャートは Lightweight Charts を使用し、シリーズごとに色分けされ凡例が表示される
  - 選択なしで反映ボタンを押すとチャートがクリアされる

### US-8.2: カスタムチャートの軸グループ制約

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、Y 軸の数が制限されていることを理解し、適切にソースを選択したい。チャートの可読性を維持するためである。
- **受け入れ条件**:
  - Y 軸は最大2種類まで（バックエンドから取得した `max_axes` に従う）
  - 同一軸グループのソースは同じ Y 軸を共有する
  - 軸グループの種類数が最大数を超える場合、反映ボタン押下時に警告メッセージが表示され、チャートは更新されない
  - 選択を変更して再度反映することで正常に描画できる

### US-8.3: カスタムチャートのデフォルト表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、カスタムチャートページを開いた時点で代表的な金融指標が表示されていてほしい。即座に有用な情報を確認するためである。
- **受け入れ条件**:
  - ページ初回ロード時に、デフォルトソース（政策金利・米国10年債利回り）が自動的にチェック状態になる
  - デフォルトソースのデータが自動的に取得され、チャートに描画される（反映ボタンを押す必要なし）
  - デフォルトソースの設定はバックエンドの JSON 定義で管理される

### US-8.4: カスタムチャートのグループ化表示

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、チェックボックスが軸グループごとにまとまって表示されてほしい。どのソースが同じ Y 軸を共有するか視覚的に把握するためである。
- **受け入れ条件**:
  - チェックボックスが軸グループごとにまとまり、グループヘッダー（日本語の表示名と単位）が表示される
  - 通常グループのヘッダーは「表示名（単位）」の形式で表示される（例: 「金利・スプレッド（%）」）
  - 通常グループ内の各ソースには単位を表示しない（グループ内で同一のため）
  - グループ内にサブカテゴリがある場合、サブカテゴリのヘッダーが表示される
  - サブカテゴリに属さないソースはグループ先頭に表示される
  - グループの表示順はバックエンドの定義順に従う
  - チャートが上、操作パネル（チェックボックス + 反映ボタン）が下に配置される
  - 警告メッセージはチャートと操作パネルの間に表示される

### US-8.5: カスタムチャートの独立軸ソース

- **アクター**: ログインユーザー
- **ストーリー**: ユーザーとして、他の指標と単位が異なるソース（投資環境スコア等）を適切にチャートに表示したい。独立した軸を持つ指標も比較に活用するためである。
- **受け入れ条件**:
  - 独立軸ソースはグループヘッダーに表示名のみ表示される（単位なし）
  - 独立軸グループ内でもサブカテゴリで分類される
  - 独立軸ソースの横にはソース固有の単位が表示される（例: `[スコア]`）
  - 独立軸ソースは選択されたソースごとに1軸を消費する（通常グループは何個選択しても1軸）
  - 独立軸ソースも軸数制約（最大2軸）の範囲内で他ソースと組み合わせて使用できる
