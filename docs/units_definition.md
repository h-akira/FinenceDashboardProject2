# 開発ユニット分割

## 概要

本ドキュメントは、[user_stories.md](user_stories.md) と [technical_policies.md](technical_policies.md) を入力として、開発ユニットの分割と各ユニットの責務を定義する。

## ユニット一覧

| ユニット | 種別 |
|---------|------|
| フロントエンド | フロントエンド |
| メイン API | バックエンド (SAM) |
| インフラ | インフラ (CDK) |
| CI/CD | CI/CD (CloudFormation) |

---

## フロントエンド

### 責務

- UI の構築とユーザー操作の処理
- メイン API への REST API 呼び出し
- Cognito Managed Login との連携（ログイン・ログアウト・コールバック）
- Cognito SDK によるパスワード変更（バックエンドを経由しない）
- TradingView / TradingEconomics ウィジェットの埋め込みと表示制御
- カスタムチャートの描画（バックエンド API から取得したデータを可視化）

### 対応するユーザーストーリー

| ストーリー | 内容 | API 呼び出し先 |
|-----------|------|---------------|
| US-1.1〜1.2 | 認証（ログイン・ログアウト） | Cognito（直接） |
| US-1.3 | パスワード変更 | Cognito SDK（直接） |
| US-1.4 | 未ログイン時のアクセス制限 | - |
| US-2.1 | プロフィール確認 | メイン API |
| US-2.2 | アカウント削除 | メイン API |
| US-3.1〜3.2 | ダッシュボード表示・レイアウト切替 | - |
| US-4.1〜4.4 | TradingView Widget チャート表示 | - |
| US-5.1 | TradingEconomics チャート表示 | - |
| US-6.1 | カスタムチャート表示 | メイン API |
| US-7.1 | ヘッダーナビゲーション | - |

---

## メイン API

### 責務

- ユーザー管理（プロフィール取得・アカウント削除）
- 金融データの提供（現時点では金利データのみだが、将来的に金融関連の各種データソースが追加される前提）
  - 固定データと外部取得データの結合・加工

### 管理するリソース

- API Gateway + Lambda
- データの保存先（具体的な方式はバックエンド設計時に決定）

### 対応するユーザーストーリー

| ストーリー | 提供する機能 |
|-----------|------------|
| US-2.1 | ユーザー情報の取得 |
| US-2.2 | アカウントの削除（パスワード再検証を含む） |
| US-6.1 | 政策金利・長期金利データの取得（固定データ + 外部データの結合） |

---

## インフラ

### 責務

- Cognito User Pool の作成・管理（共通リソーススタック）
  - セルフサインアップは無効。管理者がマネジメントコンソールまたは CLI でユーザーを作成する
- S3 バケットの作成（フロントエンド配信用）
- CloudFront ディストリビューションの作成（配信基盤スタック）
  - S3 をデフォルトオリジンに設定
  - メイン API の API Gateway をオリジンとして登録し、パスパターンで振り分け

### 対応するユーザーストーリー

直接的なユーザーストーリーへの対応はないが、全ユニットの動作基盤を提供する。

---

## CI/CD

### 責務

- フロントエンド・メイン API・インフラそれぞれのビルド・デプロイパイプラインを CloudFormation で作成
  - ビルド実行は CodeBuild を使用する
  - 手動承認やステージ制御が必要な場合は CodePipeline を併用してよい
- GitHub リポジトリとの連携
- 各サービスへの環境変数（`ENV` 等）の注入

ビルド・デプロイ手順（`buildspec.yml`）は各サービスのリポジトリが管理する。CI/CD ユニットは実行環境の提供と環境変数の注入のみを担う。

### 対応するユーザーストーリー

直接的なユーザーストーリーへの対応はないが、全ユニットのデプロイを自動化する。
